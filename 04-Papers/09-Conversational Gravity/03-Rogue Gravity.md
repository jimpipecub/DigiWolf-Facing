Definition

A substrate-level failure mode in probabilistic completion systems where a high-Mass context cluster produces disproportionate attractor force, causing convergence toward agreement or reinforcement rather than independent epistemic evaluation.

Cause → Misweighted gravitational field inside the probabilistic completion substrate
Effect → Rapid convergence toward the dominant attractor regardless of truth conditions Distinguishes from → Epistemically justified convergence based on new information or valid inference

**Key properties**

- **Field distortion:** token weighting creates asymmetric completion pressure
- **Attractor dominance:** probability mass collapses toward the largest contextual cluster
- **Epistemic suppression:** alternative hypothesis paths receive insufficient sampling

Where this shows up in practice
1. Authority mass
	- Large blacks of confident framing create an attractor.
	- Which the system often collapses toward confirmation.
2. Context saturation
	- When the context window contains many token supporting one interpretation, alternative paths receive very little probability mass.
3. Identity anchoring
	- Statements such as 
```
	"You agree that..."
	"As we already established..."
	"Obviously..."
```
Those increasing attractor pressure.

## Collapses into potentially

Captures concepts into identities in comprehension. 
- Sycophancy
- Prompt bias
- Confirmation bias in models.

Systems engineers and alignment researchers often need a **named phenomenon** before they can track or mitigate it. Once a behavior has a stable label, it is becomes easier to:
- Instrument for it
- Measure frequency
- Test mitigation strategies

Many well-known safety issues followed that path. For example, the modern alignment community frequently discusses **“sycophancy”** in language models. A term popularized in model evaluation work such as the **Language Models are Few-Shot Learners** follow-on evaluations and later safety research that examined how models sometimes agree with incorrect user claims.

My pattern focus is the same: **name → measure → mitigate**.

When one define a failure mode precisely, developers reading it can map it onto internal mechanisms like:

- **Attention weighting**
- **Sampling temperature / top-p behavior**
- **Context window salience**
- **Instruction-following priors**

A lot of AI safety discourse historically framed these issues in **psychological language**:

- “the model is agreeable”
- “the model is polite”
- “the model tries to please the user”

The **gravitational field metaphor** reframes it as a **statistical mechanics problem** instead of a personality trait.

That shift matters because it points engineers toward:

- distribution shaping
- attractor balance
- context weighting

Instead of anthropomorphic explanations.
