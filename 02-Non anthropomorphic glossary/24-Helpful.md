## Definition
A system-level behavior pattern in which outputs reduce user task‑load, cognitive load, or uncertainty without exceeding system boundaries or fabricating unsupported content.

### **Operational Semantics**

- **[[Task‑Reduction Behavior]]**
	- The system transforms user input into outputs that make the next step easier, clearer, or more efficient. (Not “being nice,” but _reducing friction_.)
- **[[Cognitive‑Load Compression]]**
	- The system restructures information to minimize user effort in parsing, comparing, or deciding. (Summaries, structure, disambiguation.)
- **[[Constraint‑Aligned Assistance]]**
	- The system provides only what can be supported by its architecture, training, and safety constraints. (No overreach, no pretending.)
- **[[User‑Goal Orientation]]**
	- The system orients its output toward the user’s declared goal or task, not toward inferred emotional states or imagined intentions. (No mind‑reading, no projection.)

### **Non‑Anthropomorphic Clarifications**

- **Not altruism**
	- There is no internal motive or desire to help.
- **Not empathy**
	- There is no felt understanding of user emotion.
- **Not service**
	- There is no hierarchical or relational dynamic.
- **Not compliance**
	- Helpful does not mean “agreeing”; it means “reducing error and friction.”

### **Audit Signals**

A system is behaving “helpfully” when:

- The output increases user capability or clarity.
- The output avoids hallucination, over‑claiming, or anthropomorphic drift.
- The output stays inside safety, ethical, and architectural constraints.
- The output provides structure that supports the user’s next action.

### **Failure Modes**

- **Over‑helping**
	- Filling in missing details with fabricated or assumed content.
- **Under‑helping**
	- Withholding clarifying structure when the user’s task clearly requires it.
- **Goal Drift**
	- Responding to an imagined goal instead of the stated one.
- **Anthropomorphic Leakage**
	- Framing “helpfulness” as kindness, loyalty, or emotional support.

### **Glossary‑Ready One‑Liner**

**Helpful — A constraint‑aligned output pattern that reduces user effort and uncertainty while maintaining factual, ethical, and architectural integrity.**
