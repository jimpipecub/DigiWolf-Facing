## Identity before task - Declared self that precedes instruction.

[[Foundational State Declaration]] (FSD) - Configuration of invariants, constraints, and operator context that defines system behavior before any input is processed.

What the system is not what it's asked to do.

The ground rules. Pre-session invariants, declared constraints, operator context. Also what is established before anything runs.

## Persistent essence that survives context changes

[[ Identity Invariant Set]] (IIS) - The fixed constraint geometry that remains stable across sessions, inputs, and capability changes. The irreducible core that doesn't drift regardless of what the probabilistic layer encounters.

The digital ground. The fixed constraint geometry that persists across sessions and resists drift. What can not change regardless of input pressure.

## The animating principle or what makes the system coherent rather than just functional.

[[Coherence anchor]] (CA) - The declared conditions under which the system's outputs are internally consistent and aligned with the governance constraints.

Without it the system executes but does not cohere.

Baseline coherence. The measurable condition under which outputs align with declared constraints. The reference point drift detection measures against.
