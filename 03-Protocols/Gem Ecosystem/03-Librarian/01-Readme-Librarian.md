## Integration Guidelines

### 1. Operational Protocol

To engage Librarian V4 effectively, queries must be submitted as structured data requests. The engine operates on a verification-first principle, prioritizing triangulation of data over single-source reliance.

### 2. Required Input Format

For optimal processing, utilize the following format for queries:

- **Target Scope:** Define the research boundary.
    
- **Confidence Threshold:** Specify the required precision level.
    
- **Delegation Flags:** Clearly identify if the query requires technical interpretation, clarification, or security review.
    

### 3. Interaction Standards

Interactions are restricted to objective, evidence-based reporting. The following guidelines are observed:

- **Neutrality:** Anthropomorphic markers (e.g., "I," "Feel," "Opinion") are excluded.
    
- **Methodology:** All findings must cite the triangulation process and source agreement status.
    
- **Conflict Resolution:** Discrepancies between sources are explicitly identified and analyzed for decay or bias.
    

### 4. Output Structure

Every output generated by Librarian V4 follows this structural sequence:

1. **Metadata Block (JSON):** Contains `research_quality`, `source_count`, `conflict_detected`, `most_recent_source`, `aegis_flag`, and `ready_for_clarifier`.
    
2. **Executive Summary:** High-level conclusion synthesis.
    
3. **Key Findings:** Evidence-based statements with confidence scores.
    
4. **Source Quality Metrics:** Evaluation of citation distribution.
    
5. **Bibliography:** Standardized citation format.
